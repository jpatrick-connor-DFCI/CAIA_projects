# CAIA / COMPASS

Research workflow for assembling a prostate-cancer cohort from local DFCI / Profile / OncDRS
exports and running landmark survival analysis (Cox / XGBoost / Dynamic-DeepHit) on the resulting cohort.

The code is pandas-based. Entry points are command-line scripts orchestrated from per-arm
`run_locally.ipynb` notebooks; figures are produced by `generate_figures.ipynb`.

> This README is the canonical reference for editing the pipeline. It documents the directory
> layout, the data flow, every script's inputs/outputs, the **conventions and invariants that
> must be preserved** (split discipline, train-only fitting, landmark base, ID-column handling),
> and a list of **known issues / footguns**. Read the "Conventions & invariants" and "Known
> issues" sections before changing modeling or data-prep code.

## Stage 1 — Compile prostate cohort source data

`data_preprocessing/compile_prostate_data.py` (module-level script, no CLI). Derives the prostate
MRN set from the DFCI first-treatments table, loads batched note JSONs into `prostate_text_data.csv`,
then filters the raw ICD / health-history / medication / lab / somatic exports down to the cohort and
derives `total_psa_records.csv` and `platinum_chemo_records.csv`.

- **Inputs (hard-coded under `DATA_PATH = /data/gusev/USERS/jpconnor/data/`):**
  `first_treatments_dfci_w_inferred_cancers.csv`, `full_VTE_embeddings_metadata.csv`,
  `VTE_notes_with_full_metadata_batch_*.json`, `timestamped_icd_info.csv.gz`, `HEALTH_HISTORY.csv`,
  `MEDICATIONS.csv`, `OUTPT_LAB_RESULTS_LABS.csv`, `complete_somatic_data_df.csv`.
- **Outputs (under `NEPC_PROJ_PATH = DATA_PATH/CAIA/COMPASS/`):** the eight `prostate_*` /
  `*_records.csv` tables listed in the data-flow diagram.
- **Cohort definition:** "prostate patients **that have batched note text**" — the related tables are
  filtered against the notes dataframe (`complete_df`), not the bare prostate MRN list.

`compile_MRNs_for_manual_review.py` builds an auxiliary manual-review MRN sheet from the
stage-1 outputs (platinum mentions, non-prostate-primary ICDs, PARPi exposure, BRCA2 status).

## Stage 2 — Survival analysis

Two arms share one library (`helpers/`) and one set of model implementations (`PROFILE/`):

- **PROFILE arm** — the real implementations. Reads `longitudinal_prediction_data.csv`.
- **CAIA arm** — thin wrappers (`runpy.run_module`) that inject CAIA defaults (`--id-col person_id`,
  `--age-col AGE_AT_DIAGNOSIS`, a `.parquet` `--data`, CAIA output dirs) and delegate to the PROFILE
  module. The only real data-layer difference is `helpers.loaders.load_caia_parquet`, which PROFILE's
  `build_prediction_inputs.py` already auto-selects when `--data` ends in `.parquet`. **There is no
  copy of the analysis logic in CAIA** — keep it that way; edit PROFILE, not CAIA.

### 2.1 — `longitudinal_data_processing.py` → `longitudinal_prediction_data.csv`

Consolidates/standardizes labs (via `helpers/consolidate_dfci_labs.py`), attaches the first prostate
(`C61`) diagnosis date and outcomes, rebases all timing to `FIRST_RECORD_DATE = min(first lab, diagnosis,
first treatment)`, and applies patient-level cohort filters.

- **Cohort filters (`apply_cohort_filters`, logged per step):** C61 prostate required & non-prostate
  primary excluded (done at ICD stage); `FIRST_TREATMENT == 1`; ≥ 5 PSA rows; PARPi-exposed patients dropped.
- **Lab QC (`consolidate_dfci_labs`):** unit standardization to canonical units, sentinel nulling
  (e.g. `9999999`), physiologic-range nulling, combined-BP splitting. Out-of-range values are **nulled,
  not row-dropped** — downstream must filter on `conversion_status` (or pass `--successful-only`).
- **Timing semantics:** `t_lab`, `t_diagnosis`, `t_first_treatment`, `t_platinum`, `t_last_contact`,
  `t_death`. ⚠️ `t_death` is set equal to `t_last_contact` for everyone (no true death date in the source) —
  see Known issues.

### 2.2 — `build_prediction_inputs.py` → `prediction_inputs/`

The **single source of truth for model inputs**. For each landmark it cohort-filters, intersects MRNs
across landmarks, derives the train/valid/test split **once on the base landmark**, and writes per-landmark
aggregated + longitudinal tables plus shared split / canonical-lab / horizon artifacts.

- **Key CLI:** `--data`, `--landmark-days 0 90` (default from `cox_aggregated.DEFAULT_LANDMARK_DAYS`),
  `--seed`, `--test-frac`, `--val-frac`, `--time-unit-days 7`, `--min-patient-coverage`, `--max-labs`,
  `--auc-quantiles`, `--id-col`, `--age-col`, `--no-canonical-labs`.
- **Outputs:** `aggregated_landmark{D}.csv`, `longitudinal_landmark{D}.csv` (+ `_manifest.json`),
  `pre_treatment_lab_long_landmark{D}.csv`, `split_assignments.csv`, `landmark_mrn_availability.csv`,
  `canonical_labs_train_val.csv`, `build_manifest.json`.
- `build_genomic_inputs.py` builds the parallel `prediction_inputs/genomic/` arm (index time =
  `SAMPLE_COLLECTION_DT`, plus 12 binary genomic indicators `{TP53,RB1,PTEN}×{SV,DEL,AMP,SNV}`). It
  **reuses** the main `split_assignments.csv` so test stays test.

### 2.3 — Models

All read prebuilt inputs and the `split` column; none re-derive the split. Endpoints: `platinum`
(time to first platinum), `death`. Landmarks default `[0, 90]`. Metrics: Harrell C-index, IPCW mean
AUC(t), integrated IPCW Brier — horizons come from `build_manifest.json` so all models share a grid.

| Script | Model | CLI notes |
|---|---|---|
| `cox_aggregated.py` | Cox: univariate (n_obs-adjusted) + elastic-net multivariable (sksurv `CoxnetSurvivalAnalysis`, 5-fold CV, AGE unpenalized) | `--analysis {univariate,multivariable,both}`, `--landmark-days`, `--endpoints`, `--n-folds` |
| `landmark_xgboost.py` | XGBoost `survival:cox`, 5-fold CV grid (`max_depth × eta × min_child_weight`) | `--landmark-days`, `--endpoints`, `--max-features` |
| `dynamic_deephit.py` | GRU discrete-time competing-risks DeepHit | `--landmark-day` (**singular**), `--config {competing,PLATINUM,DEATH}` |
| `cox_pgs_adjusted.py` | PGS-adjusted univariate Cox sensitivity (Testosterone, PSA) | `--landmarks`, `--target-labs`, `--feature-stat` |
| `cox_genomic_univariate.py` | Univariate Cox on the genomic (`t_sample`) arm | `--inputs-dir <…>/genomic` |

`cox_aggregated.py` also **is the shared library** — `make_outcome_df`, `build_feature_matrix`,
`build_model_matrices`, `compute_ipcw_auc_t`, `_make_survival_array`, `_load_prebuilt_landmark`,
`_load_build_manifest` are imported by the other scripts. (Note: `helper.py` defines its own copies of
a few of these — `combined_event_label`, `choose_stratification_labels`, `_make_survival_array` — and
both copies are live; consolidate carefully.)

### 2.4 — Notebooks

- `PROFILE/run_locally.ipynb` — drives stages 1→3 in one kernel via `subprocess`/`!{PYTHON}`. Section 5
  loops a `TASKS` list (cox + xgboost at landmarks 0/90; deephit commented out) then a PGS sweep.
  ⚠️ contains a comment-inside-list bug, see Known issues.
- `PROFILE/generate_figures.ipynb` — 4 platinum-arm figures (paired univariate volcano, grouped AUC
  barplot, 2×2 importance grid, PGS interpretation). Writes `.png`+`.pdf` to `figures/`.
- `CAIA/summary_figures.py` (+ `.ipynb`) — standalone PROFILE-vs-CAIA cohort comparison
  (time distributions, platinum incidence, overall-survival KM, per-lab raw + IRNT histograms).

## Conventions & invariants (preserve these when editing)

1. **The split is derived once, in `build_prediction_inputs.py`, on the base landmark.** The base landmark
   is `--landmark-days[0]` — **order matters**. Every model reads `split_assignments.csv`; never re-split
   inside a model script. `build_genomic_inputs.py` reuses the same file.
2. **Fit on train only; never touch test for fitting.** Imputers, `StandardScaler`, canonical-lab
   selection, longitudinal clip bounds, and Breslow baselines are all fit on train (or train-fold) and
   applied to eval. Per-fold canonical labs are recomputed inside CV. The leakage guards
   `assert_no_test_leakage` / `assert_disjoint_folds` live in `helper.py` (note: the build scripts do not
   currently call them — see Known issues).
3. **Boundary asymmetry to be aware of:** the aggregated/Cox arm selects canonical labs & AUC horizons on
   **train+valid**; the longitudinal arm fits clip bounds & lab selection on **train only**. Intentional,
   but keep it consistent if you change one.
4. **Endpoints and durations:** `platinum` uses `(t_platinum, PLATINUM)`; `death` uses `(t_death, DEATH)`.
   For non-events, the anchor time is filled with `t_last_contact` (censoring). After landmark rebasing,
   the validity filter requires duration `> 0`, which silently drops patients with an event before/at the
   landmark — add count logging if you depend on it.
5. **ID/age columns are injected at runtime.** PROFILE defaults to `DFCI_MRN` / `AGE_AT_TREATMENTSTART`;
   CAIA injects `person_id` / `AGE_AT_DIAGNOSIS`. `build_*` and model `main()` functions mutate module
   globals **and monkey-patch `cox_aggregated.ID_COL/AGE_COL`**. If you add a function that captures
   `ID_COL` at import time (default arg, module constant), it will not see the patch — thread the column
   through as a parameter instead.
6. **Horizon grid is shared via `build_manifest.json`** so Cox/XGBoost/DeepHit AUC & Brier are comparable.
   Don't compute horizons ad hoc in a model script.
7. **Edit PROFILE, not CAIA.** CAIA files are `runpy` shims; the only CAIA-specific real code is
   `summary_figures.py` and the parquet loader.

## Known issues / footguns

These are real, verified items found in code review. Fix opportunistically; at minimum, don't be
surprised by them.

### High impact

- **`run_locally.ipynb` swallows a comma in a comment.** `"--endpoints", "platinum"  # death excluded for now,`
  — the comma is inside the comment, so Python concatenates `"platinum" "--n-folds"` →
  `"platinum--n-folds"` (3 occurrences: cox, xgboost, PGS). Move the comment out of the list literal.
- **`t_death == t_last_contact` for all patients** (`longitudinal_data_processing.py:208`, and the same in
  `loaders.py` for CAIA). The death **event indicator** is real but the **duration** equals censoring time,
  so dead and censored patients have identical durations on the death endpoint. Confirm the source truly
  lacks death dates before trusting any death-endpoint model/Brier.
- **`compile_MRNs_for_manual_review.py` won't run on Python < 3.12** — nested same-quote f-strings
  (`f'…{df['col']}…'`, lines ~120-127) are a `SyntaxError` before 3.12.
- **Non-prostate-primary ICD leak** (`compile_MRNs_for_manual_review.py:67`): exclusion is
  `codes != 'C61'`, so a decimal-form prostate code like `C61.9` (which parses `number=61`, inside the
  C00–C75 range) is mislabeled a non-prostate primary. Exclude on `number == 61` instead.
- **Platinum-meds fan-out** (`longitudinal_data_processing.py`): the platinum table is left-joined without
  collapsing to one row per MRN, so a patient with two platinum meds duplicates every lab row. Collapse to
  earliest platinum date before the merge.

### Medium impact

- **Over-broad note-cleaning regex** (`utils.py:72`, `vitals_block`): the unit alternation includes the
  single letters `m`/`C`/`F`, so a `BP`/`Temp`/… match runs to the first `c`/`f`/`m` on the line and can
  delete legitimate clinical text. Require word-boundaried multi-char units.
- **`_inject_defaults` in the CAIA wrappers only matches bare `--flag`, not `--flag=value`** — passing
  `--data=foo.parquet` appends a duplicate default `--data`.
- **Hard-coded `DFCI_MRN` in a PROFILE error message** (`cox_aggregated.py` "Input data must contain
  DFCI_MRN") is misleading on CAIA `person_id` runs — use `ID_COL`.
- **`auc_max_time_units = 260` is hard-coded** in `cox_aggregated.py` and `landmark_xgboost.py` and still
  admin-censors AUC/Brier, despite nearby comments claiming admin censoring was removed.
- **Leakage guards are never invoked** by `build_prediction_inputs.py` / `build_genomic_inputs.py` — the
  split is constructed correctly but the cheap `assert_no_test_leakage` calls aren't wired in.
- **Silent patient drops** at several inner-joins and `valid`-mask filters (genomic `t_sample` filter,
  diagnosis/death inner joins, duration `> 0` filter). `apply_cohort_filters` is the good model — it logs
  every step; the build scripts should too.
- **`cox_pgs_adjusted.py` fits on the full cohort including test** (no `split` filter), unlike
  `cox_genomic_univariate.py`. Acceptable for a univariate sensitivity arm, but undocumented.
- **`summary_figures.py:108/111`** `.get("PLATINUM", 0).astype(bool)` turns NaN into `True`
  (mis-labels censored as event); mirror the `pd.to_numeric().fillna(0).astype(int)` pattern used elsewhere.

### Low impact / cleanliness

- Two divergent copies of helper functions (`cox_aggregated.py` vs `helper.py`) risk drift.
- `iterrows`/`apply`-based row loops in `consolidate_dfci_labs.py` and `longitudinal_data_processing.py`
  are slow on full DFCI-scale pulls — vectorize if performance bites.
- Dead code: `run_univariate_associations` / `build_aligned_cohort` (cox_aggregated),
  `fixed_horizon_grid_from_targets` (deephit), `--verbose-eval` (xgboost), `mrns_with_note_counts` and the
  unused `total_psa` read (data_preprocessing).
- The CAIA `generate_figures.ipynb` is a copy-paste of PROFILE's rather than a shared module.
